Development Guideline

Quy tắc bắt buộc khi phát triển trên PyWebFW: quy tắc phụ thuộc tầng, chuẩn code, quy trình thêm tính năng, checklist bảo mật.

3. Coding standards

  • Type hints ở mọi chữ ký hàm/method; from __future__ import annotations đầu file.
  • Class nhỏ, một trách nhiệm (SRP); không god-object; không file "tiện ích" hỗn tạp.
  • Đặt tên theo mẫu sẵn có: XxxService, XxxRepository, XxxPage, XxxJob, XxxApiController.
  • Field private prefix _; expose qua property/method, không expose dict nội bộ.
  • Lỗi nghiệp vụ: raise subclass của FrameworkError — KHÔNG raise HTTPException trong service/repo.
  • Log qua LoggerFactory.get(...) với fields có cấu trúc — không print().
  • Docstring cho class/method public giải thích vì sao, không kể lại code.
  • Dataclass cho value object (frozen nếu bất biến); Pydantic chỉ ở biên API.

Quy tắc dữ liệu

  • SQL luôn parameterized (?); cấm f-string nối giá trị vào SQL.
  • Cột sort phải nằm trong sortable_columns whitelist của repository.
  • Ghi nhiều bảng liên quan → bọc trong UnitOfWork.
  • Mọi output HTML/XML đi qua esc() / xml_escape — kể cả dữ liệu "chắc chắn an toàn".
  • Serialization entity nhạy cảm dùng method chuyên dụng (to_public_dict, to_safe_dict).

4. Checklist trước khi merge / deploy

#Mục kiểm tra
1pytest -q xanh toàn bộ (hiện tại 21 test); tính năng mới có test đi kèm
2Không import class cụ thể của hạ tầng ngoài bootstrap.py
3Endpoint mới: có guard đúng (RoleGuard cho admin), có phân trang nếu trả danh sách
4Hành động ghi dữ liệu có audit log kèm actor
5Không lộ secret/password/stack trace trong response và log
6Schema thay đổi: DDL idempotent (IF NOT EXISTS) + cân nhắc index
7Job mới: có retry policy hợp lý, message kết quả rõ ràng, không nuốt exception
8Chạy thử đúng chế độ module sẽ deploy (APP_MODULES)

5. Quy trình test

.\.venv\Scripts\python -m pytest -q              # toàn bộ
.\.venv\Scripts\python -m pytest tests\test_admin.py -q   # một nhóm
.\.venv\Scripts\python -m pytest -k "search" -q  # theo từ khóa
  • Test qua TestClient + ApplicationBuilder(build_test_settings(...)) — không mock tầng giữa.
  • DB test là file tạm (tmp_path) — không đụng data/app.db.
  • Test bảo mật là bắt buộc với endpoint mới: thiếu token → 401, sai role → 403, payload sai → 422.
  • Job mới: gọi engine.run_job_now(name) và assert JobStatus.SUCCESS.
