SRS — Software Requirements Specification

Đặc tả yêu cầu chức năng và phi chức năng của framework PyWebFW, phiên bản 0.1.0 phiên bản 0.2.0.

⚑ Last updated: 2026-07-22 — CASAN Maturity Audit: 18 bug fixes (correctness/security/performance), TotpProvider, 2FA TOTP, redirect module, traffic analytics, batch flush, cache hardening. 70/70 tests green.

1. Giới thiệu

1.1 Mục đích

PyWebFW là framework Python hướng đối tượng, tái sử dụng được, cung cấp nền tảng chung cho các hệ thống web cỡ vừa và lớn: trang public cho người dùng cuối, khu vực quản trị, API cho cả hai vùng, và hệ thống tác vụ tự động (scheduler/cron).

1.2 Phạm vi

Một codebase duy nhất, triển khai được theo 2 hình thái: gộp (all-in-one, 1 server) hoặc tách (FE / Admin / Scheduler chạy container riêng, có thể đặt trên server khác nhau).

1.3 Thuật ngữ

Thuật ngữĐịnh nghĩa
ModuleMột vùng chức năng triển khai độc lập: public, admin, scheduler (biến APP_MODULES)
RBACRole-Based Access Control — phân quyền theo vai trò (admin / editor / viewer)
Audit logNhật ký hành động hệ thống lưu trong DB (bảng audit_logs)
JobTác vụ tự động kế thừa BaseSchedulerJob, chạy theo lịch Interval hoặc DailyTime

6. Ràng buộc & giả định

  • Python ≥ 3.11; Docker image dùng python:3.12-slim, chạy non-root.
  • SQLite trên shared volume chỉ dùng khi các container cùng 1 host; tách nhiều máy phải chuyển sang DB server (implement BaseDatabaseManager).
  • Tài khoản seed admin / ChangeMe!123 chỉ cho lần boot đầu — bắt buộc đổi ngay.
  • Production bắt buộc đặt SECURITY_SECRET_KEY; thiếu sẽ từ chối khởi động.
  • Cache in-memory là per-process; nhiều instance cần Redis để đồng bộ invalidation.
