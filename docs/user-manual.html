User Manual

Hướng dẫn cài đặt, triển khai bằng PowerShell scripts, vận hành trang Admin và xử lý sự cố.

1. Chạy local (không Docker)

python -m venv .venv
.\.venv\Scripts\python -m pip install -r requirements.txt
copy .env.example .env        <# sửa SECURITY_SECRET_KEY #>
.\.venv\Scripts\python run.py
URLNội dung
http://127.0.0.1:8000/Trang public (Home, About, Search…)
http://127.0.0.1:8000/adminKhu vực quản trị
http://127.0.0.1:8000/api/docsSwagger UI (chỉ khi APP_DEBUG=true)
http://127.0.0.1:8000/healthzHealth probe (DB, modules, scheduler)
Tài khoản khởi tạo: admin / ChangeMe!123 — hệ thống bắt buộc đổi mật khẩu ở lần đăng nhập đầu: mọi màn hình/API admin bị chặn và tự chuyển đến /admin/change-password cho tới khi đặt mật khẩu mới. Logout hoặc đổi mật khẩu sẽ thu hồi ngay mọi token đang lưu hành của tài khoản đó.

Tham số chung (mọi script)

Tham sốMặc địnhÝ nghĩa
-Server(trống = local)Hostname/IP server từ xa — deploy qua SSH
-UserdeployUser SSH trên server
-SshPort22Cổng SSH
-RemotePath/opt/pywebfwThư mục dự án trên server
-Domain—Bật HTTPS qua Caddy: SSL Let's Encrypt tự cấp + tự gia hạn, redirect HTTP→HTTPS, HSTS

-NoBuild—Dùng lại image sẵn có, không build lại

Ví dụ

cd deploy
.\deploy-all.ps1                                     <# local, đủ 3 container #>
.\deploy-fe.ps1 -Server 10.0.0.21 -User deploy        <# FE lên server riêng #>
.\deploy-admin.ps1 -Server 10.0.0.22 -User deploy     <# Admin lên server nội bộ #>
.\deploy-all.ps1 -Server web01 -User deploy -Domain example.com   <# + HTTPS tự động #>
.\deploy-allinone.ps1 -Server web01.example.com -User ubuntu -RemotePath /srv/pywebfw

Gắn domain + SSL (production)

  • Trỏ DNS: bản ghi A của example.com và admin.example.com về IP server.
  • Deploy kèm -Domain example.com — script tự ghi DOMAIN/ADMIN_DOMAIN vào .env và bật container Caddy.
  • Caddy tự xin chứng chỉ Let's Encrypt ở request đầu tiên và tự gia hạn — không cần certbot/cron.
  • Port ứng dụng (8000/8001/8002) chỉ bind 127.0.0.1 — bên ngoài bắt buộc đi qua HTTPS của Caddy.
  • Nên bật chặn IP cho domain admin: sửa deploy/Caddyfile (block @outside có sẵn, chỉ cần bỏ comment và điền dải IP văn phòng/VPN).
Deploy remote yêu cầu: máy chạy script có OpenSSH client + tar (Windows 10+ có sẵn), server đích có Docker + docker compose và SSH key đã trao đổi (không hỗ trợ password prompt).

4. API quick reference

Mọi response theo envelope chuẩn: {"success": bool, "data": …, "error": {code, message}, "meta": {page, size, total, pages}}.

Đăng nhập lấy token (API client)

POST /api/admin/auth/login
Content-Type: application/json

{"username": "admin", "password": "•••"}

→ data.token  → dùng header:  Authorization: Bearer <token>
MethodEndpointGhi chú
GET/api/public/menus · /content/{slug} · /search?q= · /sitemapKhông cần auth
POST/api/admin/auth/change-passwordBody: current_password, new_password (≥8 ký tự) — trả token mới, token cũ bị thu hồi
GET/POST/api/admin/usersPaging: ?page=&size=&sort_by=&sort_desc=
GET/PUT/DELETE/api/admin/users/{id}PUT bỏ trống password = giữ nguyên
GET/POST · PUT/DELETE/api/admin/menus[/{id}]Sửa menu tự invalidate cache
GET/POST · GET/PUT/DELETE/api/admin/contents[/{id}]CRUD nội dung CMS; slug pattern a-z0-9-; tự invalidate cache slug cũ + mới
GET/api/admin/logs?level=errorlevel: info | warning | error
GET/api/admin/dashboard/metrics · /api/admin/system/health—
GET/POST/DELETE/api/admin/system/db-connections[/{id}]DSN trả về luôn được mask

5. Xử lý sự cố

Triệu chứngNguyên nhân thường gặpCách xử lý
Container không lên, log báo SECURITY_SECRET_KEY must be setThiếu .env khi APP_ENV=productionTạo .env cạnh docker-compose.yml; script deploy tự sinh nếu chưa có
401 liên tục dù vừa đăng nhậpToken hết hạn (mặc định 3600s) hoặc đổi SECRET_KEY làm token cũ vô hiệuĐăng nhập lại; client API tự refresh khi gặp 401
429 Too Many RequestsVượt rate limit (mặc định 120 req/60s/IP)Tăng RATELIMIT_MAX_REQUESTS trong env nếu là traffic hợp lệ
/admin trả 404Container đó không bật module adminKiểm tra APP_MODULES — vào đúng cổng container admin (8002)
Job báo FAILED trong dashboard/logsXem cột detail của audit log job.failedChạy thử lại: job tự retry theo policy; sửa nguyên nhân gốc (đĩa đầy, DB khóa…)
database is locked thoáng quaNhiều container ghi đồng thời (SQLite)Đã có busy_timeout 5s; nếu thường xuyên → chuyển Postgres (xem Architecture §3)
Deploy remote dừng ở bước sshChưa trao đổi SSH keyssh-keygen + ssh-copy-id user@server rồi chạy lại
